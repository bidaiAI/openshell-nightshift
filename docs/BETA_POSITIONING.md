# OpenShell NightShift — Beta Positioning

## One-line description

A privacy-first task marketplace for operators and workers, shipped today as a small-scale runnable beta.

## Product category

NightShift is not a generic tool-calling layer and not yet a full decentralized compute market.

It is best described as:

> a confidential task marketplace with controlled worker execution and verifiable delivery.

## What makes it different

- private and selective task visibility
- sealed worker bids
- receipt-based delivery
- selective reveal during disputes
- Compact-style public/private state modeling

## Current go-to-market framing

Use:

- small-scale beta
- limited beta
- deployable product
- controlled execution topology
- self-hostable reference implementation

Avoid:

- fully decentralized network
- finished production system

## Honest current state

NightShift is:

- runnable locally
- deployed publicly
- suitable for limited evaluation and small-scale testing

NightShift is not yet:

- final production auth
- final production persistence
- final decentralized worker mesh
- final on-chain settlement layer
